# Review Checklist — Single-Voice and Cross-Document Consistency

The final pass. The prose checklist catches local faults; the conformance checklist catches
missing content. This checklist answers the guide's defining question: **does this document
— and this set of documents — read as though one entity wrote it for one audience?**

Run it on a complete document, and again across a set of documents from the same entity.

---

## A. One speaker (within a document)

- [ ] **The register is constant** from first section to last — no section is suddenly
  chattier, more formal, or more promotional than the rest.
- [ ] **No first or second person anywhere**; the impersonal speaker never slips.
- [ ] **Obligation is carried only by shall/should/may**, uniformly; no section drifts into
  "must" or emphasis-as-requirement.
- [ ] **No section narrates its own production** or addresses the reader directly.
- [ ] **The overview does not "sell"**; it states facts at the same register as the views.

## B. One audience (within a document)

- [ ] **The level of explanation is uniform** — standard concepts are assumed known;
  EoI-local terms are identified on first use; no section drops into tutorial mode or
  assumes insider knowledge the rest of the document does not.
- [ ] **Every abbreviation is expanded once, early**, and used consistently after.
- [ ] **No condescension, no mystification** — neither "simply/obviously" nor unexplained
  jargon used to impress.

## C. One vocabulary (within a document)

- [ ] **Each concept has exactly one word.** The same component is not "the Settlement
  service" in one section and "the settlement engine" in another.
- [ ] **Defined terms used precisely and consistently**; no drift between *view* and
  *viewpoint*, *concern* and *requirement*, *aspect* and *concern*.
- [ ] **EoI-local glossary matches usage** — every local term defined once and used as
  defined.

## D. One structure (within a document)

- [ ] **Sections follow the canonical order and wording** of
  [`../docs/05-document-structure.md`](../docs/05-document-structure.md).
- [ ] **Identifiers follow one scheme** (SH-, C-, A-, VP-, V-, VC-, COR-, M-, ADR-) and are
  used consistently in prose and tables.
- [ ] **Tables and figures use one caption/legend pattern** throughout.
- [ ] **Decision records all use the one template**; rationale is structured the same way
  in each.

## E. One entity (across a set of documents)

Run this when reviewing more than one AD from the same authoring entity.

- [ ] **Term casing and abbreviation expansions match** across documents.
- [ ] **Heading scheme and section order match** across documents.
- [ ] **Decision-record format matches** across documents.
- [ ] **Standard and cross-reference citation style match** (e.g. always "42010:2022, 6.6").
- [ ] **Shared EoI-local terms agree** — a term defined in one document is not redefined
  differently in another.
- [ ] **The same register and reading level** hold across the set; no document reads as a
  different author's work.
- [ ] **A reader who learned where decisions / concerns / views live in one document finds
  them in the same place in every document.**

## F. The "blind author" test

Read three randomly chosen paragraphs from different sections (or different documents). If a
reader could correctly guess they were written by **one author for one reader**, the
document passes. If any paragraph reads as a different writer — looser, more promotional,
more personal, more technical-show-off — locate the drift and bring it back to the single
voice.

---

### Verdict

Record one of: **consistent** (passes A–F), **consistent with fixes** (list them), or
**inconsistent** (the single-voice property is not met; summarize where the voice, audience,
vocabulary, or structure diverges and what must change).
